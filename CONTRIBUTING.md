# Contributing

## How to Contribute

1. Fork the repo
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Test your templates by actually using them in a Claude Code session
5. Submit a PR with a description of what you changed and why

## What We're Looking For

- New layer patterns that solved real problems in your workflow
- Improvements to existing templates based on production usage
- Examples from different tech stacks and team sizes
- Bug fixes in bash commands or template formatting

## Guidelines

- Templates should be copy-paste ready. No placeholder paths that require editing to work.
- Keep examples anonymized. No company names, real people, or proprietary details.
- Every bash command should be tested. Broken commands in templates are worse than no commands.
- Match the existing voice: practical, direct, builder-first. No marketing language.

## Reporting Issues

Open an issue describing:
- What you tried (which template/layer)
- What happened vs. what you expected
- Your setup (Claude Code version, OS, number of terminals)
